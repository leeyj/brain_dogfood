# 🧠 뇌사료(Memo Server) 정식 기능점검 보고서 (v1.0)

사용자 증가에 따른 시스템 안정성 및 보안성을 확보하기 위해 소스코드 기반으로 도출된 정식 기능점검 상세 리포트입니다. 이 리포트는 [test_cases.csv](file:///c:/project/my_util/memo_server/docs/QA/test_cases.csv)에 정의된 50여 개의 테스트 항목을 체계적으로 수행하기 위한 가이드를 제공합니다.

## 1. 개요 (Executive Summary)
본 점검은 '뇌사료' 서비스의 핵심 기능인 지식 기록, 보안 암호화, 데이터 시각화 및 모바일 대응력을 정식으로 검증하는 것을 목적으로 합니다. 특히 사용자가 늘어남에 따라 발생할 수 있는 동시성 문제, 권한 부족 상황(Edge Case), 대용량 데이터 로드 시의 성능 저하를 집중적으로 점검합니다.

## 3. 상세 기능별 점검 가이드 (Functional Details)

### 3.1 인증 및 세션 관리 (Auth & Session)
> [!IMPORTANT]
> **핵심 검증 항목**: 로그인 유효 기간이 만료되었을 때 사용자의 데이터가 보호되는가?

- **예외 상황 테스트**:
  1. 브라우저 탭을 여러 개 띄우고 한 곳에서 로그아웃 시 다른 탭의 API 동작 차단 여부 확인.
  2. 세션 타임아웃 1분 전 알림 발생 시 마우스 클릭을 통해 세션을 연장할 수 있는지 확인.
  3. 로그인 페이지에서 임의의 SQL Injection 쿼리 입력 시 차단 여부 확인.

### 3.2 메모 레이아웃 및 성능 (Layout & Grid)
- **Masonry 고도화 테스트**:
  1. 약 1000자 이상의 긴 텍스트와 이미지가 섞인 메모들이 그리드 엉킴 없이 정렬되는가?
  2. 브라우저 창을 빠르게 좌우로 늘리고 줄일 때, 디바운스(Debounce) 로직이 작동하여 성능 저하 없이 재배치되는가?
  3. 무한 스크롤 도중 네트워크가 끊겼을 때 로딩 중단 및 재시도 UX가 제공되는가?

### 3.3 보안 및 암호화 (Security & Encryption)
- **자산 보호 테스트**:
  - 암호화된 메모의 파일 URL(예: `/api/download/uuid.jpg`)을 직접 주소창에 입력했을 때, 세션이 없으면 `403 Forbidden`이 발생하는지 확인.
  - 소스코드(`MemoService.py`) 내에서 해독 없이 업데이트 시도 시 서버 레벨에서 차단 로직이 견고한지 확인.

### 3.4 파일 관리 (File Management)
- **파일 무결성**:
  - 파일 본문은 서버 저장 시 AES-256 등으로 암호화되어 저장되는가? (물리 파일을 열었을 때 내용을 볼 수 없어야 함)
  - 대용량 파일(.zip 등) 업로드 시 서버 타임아웃 설정에 걸리지 않고 정상적으로 기록되는가?

## 4. 수행 기록 및 판정 기준 (Acceptance Criteria)

- **PASS**: 기대 결과와 실제 결과가 100% 일치하며 예외 메시지가 명확할 때.
- **CONDITIONALLY PASS**: 기능은 작동하나 소폭의 UI 밀림이나 UI 메시지 오타가 있을 때.
- **FAIL**: 데이터 손실, 보안 침해, 서비스 중단(Crash)이 발생할 때.
