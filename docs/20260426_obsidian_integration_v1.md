# 옵시디언 연동 1차 구현 결과 보고 (2026-04-26)

## 1. 개요
뇌사료(Brainsryo) 서버와 옵시디언(Obsidian) 간의 데이터 연동을 위한 전용 플러그인 1차 개발 완료.

## 2. 주요 구현 기능
- **서버 API 확장**: 
    - `GET /api/external/memos`: 동기화용 메모 목록 및 상세(마크다운) 제공.
    - `POST /api/external/memos`: 신규 메모 생성 및 ID 발급.
    - `POST /api/external/memos/<id>`: 기존 메모 업데이트.
- **옵시디언 플러그인**:
    - **Sync All**: 전체 메모 내려받기 및 이미지/링크 치환.
    - **Push Current**: 현재 활성화된 메모를 서버로 업로드.
    - **Smart Creation**: ID 없는 파일 업로드 시 ID 발급 및 파일명 자동 변경.
    - **Security Shield**: 암호화된 메모 감지 시 수정 방지(읽기 전용) 및 업로드 차단.

## 3. 기술적 특이사항
- **Frontmatter 기반 관리**: `id`, `title`, `is_encrypted` 속성을 활용한 데이터 매핑.
- **RegExp 치환**: 서버 주소(`api/download`)와 옵시디언 링크(`![[]]`) 간의 자동 변환 로직.
- **자동 배포 스크립트**: `build_and_deploy.ps1`을 통한 신속한 테스트 환경 구축.

## 4. 현재의 한계점 (Clumsy Points)
- **수동 동기화**: 매번 사용자가 'Push'를 명시적으로 실행해야 함.
- **암호화 제약**: 암호화된 데이터는 옵시디언 내에서 활용이 거의 불가능함.
- **첨부파일 역전송 불가**: 옵시디언에서 추가한 이미지를 서버로 전송하는 기능 미구현.
- **UI/UX 미흡**: 동기화 상태 표시나 에러 피드백이 단순 알림(Notice)에 의존함.

## 5. 향후 고도화 아이디어
1. **실시간 백그라운드 동기화**: 파일 변경 감지 시 자동 업로드.
2. **플러그인 내 복호화**: 비밀번호 입력 시 로컬에서 암호화 메모 수정 가능하게 구현.
3. **첨부파일 양방향 동기화**: 로컬 이미지 추가 시 서버 업로드 및 URL 획득.
4. **모바일 최적화**: 폰에서도 옵시디언을 통해 메모를 관리할 수 있는 수준의 안정성 확보.
