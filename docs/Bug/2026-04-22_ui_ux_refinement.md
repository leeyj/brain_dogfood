# 🐛 버그 조치 리포트: UI 아키텍처 고도화 및 UX 최적화 (v5.0+)

## 1. 발생 현황
- **단축키 충돌**: 기존 `Ctrl + Shift` 조합이 브라우저 기본 기능(창 닫기 등)과 충돌하여 사용자 데이터 유실 위험 발생.
- **UI 일관성 결여**: 컴포넌트화 과정에서 인라인 스타일 누락으로 인해 암호화 블록 및 ID 배지(#ID)의 시인성 저하.
- **자동 완성 간섭**: 크롬 브라우저가 검색창을 로그인 필드로 오해하여 사용자 ID를 강제 입력하는 현상.
- **편의 기능 부재**: 단축키가 많아짐에 따라 사용자가 이를 즉시 확인할 수 있는 방법이 없음.

## 2. 조치 사항
### A. 단축키 시스템 개편 (ShortcutManager.js)
- **Alt 기반 조합 도입**: 브라우저와 충돌 없는 `Alt + W`(주간), `Alt + A`(전체), `Alt + N`(신규) 단축키 추가.
- **e 키 즉시 수정 복구**: `MemoCard` 컴포넌트에서 `hoveredMemoId`를 추적하도록 이벤트를 바인딩하여 즉시 수정 기능 정상화.

### B. UI/UX 정교화 및 복구
- **Stable UI 복원**: `brain_dogfood_stable` 폴더의 원본 코드를 대조하여 암호화 블록의 보라색 강조 버튼 및 "암호화됨" 문구 복구.
- **ID 배지(#ID) 수정**: 누락되었던 `rgba(255,255,255,0.15)` 투명도 및 절대 위치(`absolute`) 스타일을 적용하여 시각적 노이즈 제거.
- **검색창 자동 완성 방지**: 검색창 ID를 `sb_query_v5`로 변경하고 `autocomplete="one-time-code"` 트릭을 적용하여 크롬의 강제 입력을 차단.

### C. 신규 기능 추가
- **단축키 도움말 모달**: 사이드바에 `❓` 버튼 추가 및 검색창에 `?` 입력 시 즉시 호출되는 커맨드 기능 구현.

## 3. 향후 주의 사항
- **컴포넌트화 시 스타일 보존**: JS 클래스로 UI를 분리할 때 `stable` 폴더의 인라인 스타일이 누락되지 않도록 철저한 대조 검수 필요.
- **브라우저 충돌 모니터링**: 새로운 단축키 조합이 특정 브라우저(특히 모바일 환경)에서 다른 기능과 겹치는지 지속적인 확인 필요.
