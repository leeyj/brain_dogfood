# 업데이트 알림 및 원클릭 업데이트 시스템 도입

## 변경 내용
- **버전 관리 체계 구축**: `version.json`을 통해 로컬 및 원격 버전을 관리하고 히스토리를 추적할 수 있도록 설계.
- **백엔드 API 신설**: `app/routes/system.py`를 통해 GitHub의 최신 버전을 체크하고 `git pull`을 실행하는 엔드포인트 구현.
- **프론트엔드 연동**: `VersionManager.js`를 통해 앱 기동 시 업데이트 여부를 감지하고, 사이드바 설정 아이콘에 알림 배지를 표시.
- **업데이트 로그 UI**: 신규 변경 사항을 확인하고 즉시 업데이트할 수 있는 전용 모달 추가.

## 조치 사항
1. **의존성 최소화**: 외부 라이브러리 없이 `urllib.request`를 사용하여 원격 버전을 조회하도록 구현.
2. **보안 강화**: 업데이트 실행 API(`POST /api/system/update`)에 `@login_required`를 적용하여 인증된 관리자만 실행 가능하도록 제한.
3. **UX 최적화**: 업데이트 성공 시 서버 재시작 안내 및 페이지 새로고침 로직 포함.

## 향후 주의 사항
- `git pull`을 통한 업데이트는 서버 환경에 `git`이 설치되어 있고 브랜치가 `main`인 경우에만 정상 작동함.
- Docker 환경에서는 `git pull` 대신 이미지 교체가 권장될 수 있으나, 현재의 네이티브 배포 환경(your-server-ip)에서는 유효하게 작동함.
- 향후 업데이트 후 자동으로 서버가 재시작되도록 `os.execv` 기반의 리스타트 로직을 보강할 수 있음.
