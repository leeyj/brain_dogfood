# 버그 조치 및 기능 개선 보고서 (2026-04-22)

## 1. 개요
메모 서버의 UI 아키텍처를 슬롯 기반으로 전환하고, 고밀도 리스트 뷰 및 주간 관리 기능을 추가함.

## 2. 버그 내용 및 조치 사항

### A. Visual Linker 좌표 불일치 버그
- **내용**: 리팩토링 후 `Alt+클릭` 드래그 시 선이 (0,0)에서 시작하거나 엉뚱한 위치에 그려지는 현상 발생.
- **조치**: 전역 이벤트 위임 방식으로 로직을 전환하고, 마우스 이벤트 객체를 동기화하여 정확한 DOM 좌표를 계산하도록 수정함.

### B. 필터링 초기화 미흡 (전체지식 미표시)
- **내용**: 특정 날짜 필터가 걸린 상태에서 사이드바의 '전체지식'을 클릭해도 날짜 필터가 유지되어 메모가 보이지 않음.
- **조치**: `AppService.setFilter`에서 그룹 변경 시 모든 날짜/범위 필터를 강제 초기화(`null`)하도록 로직을 보강함.

### C. API 쿼리 파라미터 "null" 문자열 전달
- **내용**: `AppService`에서 필터 리셋 시 `null`을 전달하면 브라우저가 이를 `"null"` 문자열로 변환하여 서버에 요청함.
- **조치**: `api.js` 전처리 단계에서 `null`/`undefined` 값을 빈 문자열(`''`)로 변환하도록 수정함.

### D. 크롬 자동 완성 방해
- **내용**: 크롬 브라우저가 검색창 및 작성기 필드에 부적절한 자동 완성 값을 채워넣어 사용자 혼란 가중.
- **조치**: `autocomplete="off"` 및 `spellcheck="false"` 속성을 주요 입력 필드에 적용함.

## 3. 신규 기능 추가
- **주간 매니저(Weekly Manager)**: 검색창 하단 주간 선택기 구현 (토글 기능 포함).
- **리스트 레이아웃(List Layout)**: 가로형 고밀도 리스트 뷰 도입.
- **주간/전체 토글 버튼**: 마우스 동선 최적화를 위해 주간 선택기 내에 스마트 토글 버튼 배치.

## 4. 향후 주의사항
- 레이아웃 추가 시 `LayoutManager`의 인터페이스를 준수할 것.
- 전역 이벤트 리스너 추가 시 `VisualLinker`와 같이 클린업 또는 중복 등록 방지 로직을 반드시 확인해야 함.
