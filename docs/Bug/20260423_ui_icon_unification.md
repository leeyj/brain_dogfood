# 버그 조치 보고서: UI 아이콘 및 라벨 불일치 해결

## 버그 내용
- **문제점**: 도움말 모달(단축키 도움말)에서 사용하는 이모지와 라벨이 실제 사이드바 메뉴에서 사용하는 아이콘(FontAwesome) 및 라벨과 서로 달라 사용자에게 혼선을 줌.
- **예시**:
    - 도움말: 🌐 전체 지식 vs 사이드바: ⊞ 모든 메모
    - 도움말: 📌 고정 메모 vs 사이드바: ★ 고정 메모
    - 도움말: 🕸️ 지식 성단 vs 사이드바: 🕸️ 지식 맵 보기

## 조치 사항
1.  **아이콘 통일**: 도움말 모달의 텍스트 기반 이모지를 사이드바와 동일한 FontAwesome 아이콘(`fas fa-th-large`, `fas fa-star` 등)으로 교체.
2.  **라벨 통일**: 도움말 모달의 하드코딩된 텍스트를 사이드바에서 사용하는 i18n 키(`menu_all_memos`, `nav_nebula` 등)로 교체하여 용어 통일.
3.  **스타일 및 구조 개선**: `modals.css`에 아이콘 정렬 및 입체적인 키캡(`kbd`) 스타일을 추가하고, 하단 탭 시스템(Excel Style)을 위한 스타일 정의.
4.  **컴포넌트화**: `HelpModal.js` 클래스를 생성하여 도움말 로직을 모듈화하고, 하단 탭 전환 기능을 통해 향후 확장이 용이하도록 구조화.
5.  **강조색 적용**: 지식 탐색기(Explorer) 및 지식 맵(Nebula) 아이콘에 사이드바와 동일한 강조색(`--accent`, `--ai-accent`) 적용.

## 향후 주의사항
- 새로운 메뉴나 기능을 추가할 때, 사이드바와 도움말 모달 양쪽의 아이콘과 라벨이 항상 동기화되도록 주의해야 함.
- 가급적 하드코딩된 텍스트 대신 `data-i18n` 속성을 사용하여 언어 팩에서 일괄 관리하도록 구성할 것.
