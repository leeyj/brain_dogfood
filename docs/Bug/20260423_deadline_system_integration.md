# 기능 추가 및 구조 개선 리포트 (2026-04-23)

## 1. 개요
단순 기록 중심의 메모 시스템에 '기한(Deadline)' 개념을 도입하여 실행 중심의 지식 관리 도구로 확장함.

## 2. 변경 내용 및 조치 사항
### A. 기한 관리 시스템 통합
- **DB 확장**: `memos` 테이블에 `due_date` 컬럼 추가 및 마이그레이션 적용.
- **작성기(Composer) 개선**: 날짜 선택기(Date Picker) 추가 및 슬래시 명령어(`/기한`) 연동.
- **시각화**: `MemoCard`에 D-Day 배지 및 상태별(Blue, Orange, Red) 테두리 강조 적용.

### B. 주간 달력(Weekly View) 고도화
- **레드 배지**: 마감일인 날짜에 펄스 애니메이션이 적용된 레드 도트 표시.
- **미래 필터링**: 미래 날짜 클릭 시 해당 날짜가 마감인 메모를 필터링하여 출력하도록 백엔드/프론트엔드 로직 수정.
- **레이아웃 최적화**: 노트북 환경을 고려하여 상하 여백 및 폰트 크기를 약 30% 이상 압축한 '울트라 콤팩트' 모드 적용.

### C. 아키텍처 리팩토링 (Modularization)
- `MemoCard.js`의 비대화 해결을 위해 `DeadlineManager.js`(로직 분리) 및 `MemoCardParts.js`(UI 조각 분리)로 모듈화.
- 코드 가독성 향상 및 유지보수 효율 증대.

## 3. 향후 주의사항
- **날짜 형식**: `due_date`는 `YYYY-MM-DD` 형식을 엄격히 준수해야 함.
- **의존성**: `MemoCard.js` 수정 시 `MemoCardParts.js`와의 인터페이스 정렬 확인 필요.
- **서버 동기화**: `deploy.py`를 통해 홈 서버(your-server-ip)에 성공적으로 배포됨.

## 4. 버그 조치
- 날짜 선택기 박스 크기 오류 및 브라우저 기본 아이콘 간섭 문제 해결.
- 특정 조건에서 격자선이 굵게 표시되던 시각적 버그 수정.
