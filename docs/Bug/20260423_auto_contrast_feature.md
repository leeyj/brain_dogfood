# [Feature/Bug] 지능형 자동 대비 (Auto-Contrast) 및 가독성 개선

## 1. 개요 및 배경
- **문제점**: 사용자가 메모 카드 배경색을 흰색(#FFFFFF) 등 밝은 색으로 설정할 경우, 기존의 밝은 텍스트 컬러(--text: #f8fafc)가 보이지 않는 가독성 문제 발생.
- **요청 사항**: 배경색에 맞춰 글자색도 사용자가 지정하게 하거나 시스템이 자동으로 맞춰줄 것을 요청함.
- **결정 사항**: 사용자의 편의성을 위해 배경 밝기를 감지하여 최적의 대비 색상을 자동 적용하는 'Auto-Contrast' 로직을 도입함.

## 2. 조치 사항
### 🎨 CSS 및 변수 체계 개선
- `static/css/components/memo.css`:
    - `.memo-card`의 배경색을 하드코딩된 값에서 `var(--card)` 변수로 전환.
    - 카드 내부 텍스트 및 제목 컬러를 `var(--text)`로 통일하여 유동적 대응이 가능하게 함.
    - 하단 페이드 아웃 효과(`::after`)의 그라데이션 끝점을 `var(--card)`로 변경하여 배경색과 일치시킴.
- `static/js/components/MemoCard.js`:
    - 우측 상단 `#ID` 배지의 색상을 하드코딩된 `rgba(255,255,255,0.15)`에서 `var(--muted)`와 `opacity` 조합으로 변경.

### ⚙️ ThemeManager 로직 고도화
- `getContrastColor(colorStr)` 함수 추가:
    - 입력된 색상(HEX, RGBA)의 밝기(YIQ Luma)를 계산.
    - 밝기 임계값(128)을 기준으로 어두운 배경에는 밝은 텍스트(#f8fafc), 밝은 배경에는 어두운 텍스트(#0f172a)를 반환.
- `applyTheme(settings)` 수정:
    - 카드 배경색(`card_color`) 변경 시 즉시 대비 색상을 계산하여 `--text`, `--muted` CSS 변수 업데이트.
    - 에디터(ToastUI) 테마 연동 로직 추가 (밝기에 따라 `toastui-editor-dark` 클래스 토글).

## 3. 향후 주의사항
- **에디터 테마**: ToastUI 에디터는 초기화 시점에 테마가 결정되므로, 실시간 테마 전환 시 내부 플러그인이나 스타일이 완벽하게 대응하지 않을 수 있음. 필요 시 페이지 새로고침을 권장할 수 있음.
- **기타 요소**: 태그 배지(`tag-badge`)나 버튼들의 대비도 추후 점검 필요. 현재는 범용적인 대비 색상만 우선 적용됨.
