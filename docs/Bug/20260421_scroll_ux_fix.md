# [Bug Report] 무한 스크롤 레이아웃 흔들림 및 스크롤 점프 현상

## 버그 내용
- **현상**: 메모 서버의 메인 화면에서 무한 스크롤을 통해 새로운 메모를 불러올 때, 기존에 렌더링된 메모 카드들의 위치가 위아래로 크게 요동치거나 스크롤 위치를 잃어버리는 현상 발생.
- **원인**: 
    1. Flex 기반의 Masonry 흉내 방식에서 새로운 요소가 추가될 때 전체 그리드가 재계산됨.
    2. 레이아웃 엔진이 불안정하여 비어 있는 공간을 채우는 과정에서 레이아웃 시프트(Layout Shift)가 심하게 발생.
    3. `ui.js` 내에 기존 Flex 로직과 새로운 Masonry 로직이 중복/혼재되어 충돌 발생.

## 조치 사항
- **해결 방안**: 
    1. **수직 컬럼 기반 Masonry 도입**: 화면 너비에 맞춰 250px 너비의 수직 컬럼(`.masonry-column`)들을 미리 생성.
    2. **최적 배치 알고리즘**: 새로운 메모 카드를 추가할 때, 현재 자식 요소가 가장 적거나 높이가 가장 낮은 컬럼을 찾아 `appendChild` 하도록 변경.
    3. **JS 코드 정리**: `ui.js`에서 중복된 `appendChild` 호출 및 깨진 구문을 제거하고 Masonry 전용 로직으로 단일화.
    4. **너비 고정**: 사용자 요청에 따라 카드 너비를 250px로 고정하여 시각적 일관성 확보.

## 향후 주의 사항
- 브라우저 리사이즈 시 성능 저하를 방지하기 위해 실시간 재배치는 제외하였으나, 필요시 디바운스(Debounce) 처리를 적용한 리사이즈 리스너 검토 필요.
- 카드 내부의 이미지 로딩 등으로 인해 높이가 뒤늦게 결정되는 경우를 대비하여, 이미지에 `aspect-ratio`를 적용하거나 사전에 높이를 확보하는 방안 고려.
